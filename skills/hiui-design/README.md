# hiui-design-skill

This is a sanitized public-edition monorepo generated from the internal `hiui-design` Codex skill.

## Scope

This repository contains generic HiUI design rules, page archetypes, schemas, templates, validation helpers, public machine CLI contracts, generic typical-page examples, the public `@hiui-design/typical-page-shells` runtime package source, and the vendored runtime tgz required for direct-use installs.

It intentionally excludes private publishing automation, internal usage telemetry, internal Git/global-sync workflows, business-line overlays, private Figma IDs, and generated release outputs.

## External Use

External users can clone or download this repository and use it directly. The checked-in vendored `vendor/hiui-design-typical-page-shells-*.tgz` file is part of the default install contract, so the standard bootstrap path does not assume that `@hiui-design/typical-page-shells` has already been published to public npm.

For target-project setup, use `scripts/bootstrap-target-project.mjs` from this repository. It writes the runtime dependency as a project-local `file:` spec that points at `.local-context/hiui-design/vendor/...`, and it also adds any required host-side direct dependencies declared in `docs/onboarding/host-integration-dependencies.json`.

If the target project declares `vite@5` or newer, run bootstrap, install, dev, and build under `Node >=18`. The public bootstrap/doctor flow now reports this explicitly so external users do not hit a late `vite build` failure on Node 16.

For `host-integration`, the public bootstrap path now also provisions `src/translation/*` by default because the shipped app-frame and example pages already depend on that bridge. `rules-only` and `legacy-host-compatible` remain opt-in via `--init-i18n`.

## Machine CLI Contracts

The public contract surface is listed in `scripts/public-cli-contracts.json` under the `machine-public` tier. These selected scripts and checked-in fixtures are the stable machine-readable interface for downstream tests and agents.
