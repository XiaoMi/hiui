# 项目图片集约束

这份文件只回答一件事：

- `hiui-design` 生成页面时，只要命中商品图、设备图、媒体缩略图、产品占位图，就必须优先复用项目内的统一图片集，而不是临时生成新图。

## 项目级图片事实源

- 优先查找项目内显式图片注册表，例如：`src/**/project-product-images.ts`
- 优先查找项目内统一图片目录，例如：`src/**/product-catalog/`
- 优先复用稳定运行时入口，例如：`getProjectProductImage()`
- 优先复用稳定类型入口，例如：`imageType`、`projectProductImageCatalogEntries`
- 实际项目可以不是这些固定命名，但在生成带商品图的业务页前，必须先存在一个明确的“项目图片事实源”
- `setup-for-designers.mjs` / `apply-in-current-project.mjs` 默认会初始化一套压缩后的默认图片包与注册表骨架；若项目已有真实图片，保持原文件优先，不会静默覆盖

## 生成时的默认规则

- 只要页面出现商品图片、设备图片、产品卡片缩略图、订单商品图、资产台账设备图，就先查项目内图片注册表
- 若项目已经存在统一图片集，直接复用；页面数据层声明 `imageType`，页面组件只通过注册表入口取图
- 若当前图片集里还没有该类目，先补图片类型与注册表，再生成业务页面；不要先在业务页里塞临时图
- 若项目还没有建立真实图片集，则默认允许页面“不显示图片”或“隐藏图片区”，但不允许临时生成假图
- 若图片是该页面的核心信息而不是装饰信息，且项目还没有真实图片集，先停下并要求补图片资产，不要绕过

## 当前允许的图片来源层级

1. 项目内真实商品图 / 设备图
2. 无图

## 明确禁止

- 在业务页 `.tsx`、`.jsx`、`.ts`、`.js` 或 mock 数据中直接拼接 `data:image`
- 在单页里临时 import 一张没有纳入注册表管理的商品图
- 因为图片集暂时缺类目，就在页面局部回退到随机插画、品牌无关示意图、纯色占位块或助手生成的预制图
- 因为缺少真实图片，就重新生成一套商品插画、设备 SVG 或媒体占位图

## 生成时输出要求

若页面命中图片使用场景，结果中至少显式写出：

- `image source strategy`
- `selected image types`

默认口径：

- `image source strategy = project image catalog`
- `selected image types = ...` 列出当前页面实际使用的 `imageType`

## 建立或扩充图片集的顺序

1. 先收集真实图片资产
2. 再放入项目统一资产目录
3. 再更新项目图片注册表
4. 最后才允许在业务页里引用

不要绕过这四个步骤。
