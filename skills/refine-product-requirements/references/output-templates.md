## 需求细化摘要模板

```markdown
## 需求细化摘要

### 当前理解
- 产品类型：
- 产品目标：
- 目标用户：
- 核心场景：
- MVP 边界：
- 关键约束：
- 非目标：

### 方案概览
- 产品定位：
- 核心流程：
- 功能模块：
- 数据对象：
- 业务规则：
- 成功指标：

### 假设与待确认
| 编号 | 问题 | 推荐选项 | 可选选项 | 影响范围 | 建议确认时机 |
|---|---|---|---|---|---|
| Q1 |  | A. 方案（推荐） | B. 方案 / C. 方案 / D. 其他/自定义 |  |  |
```

若当前交付模式是 `quick-refine`，摘要模板应收口为：

- 当前理解
- 关键假设
- 2-3 个最高影响待确认问题
- 推荐下一步

不要把 `quick-refine` 输出扩写成正式方案、PRD 骨架、页面清单或页面提示词。

## 选项式确认模板

```markdown
## 待确认选项

1. <问题>
   - A. <推荐方案>（推荐）：<对范围、规则、页面或交付物的影响>
   - B. <备选方案>：<影响>
   - C. <备选方案>：<影响>
   - D. 其他/自定义：<需要用户补充的内容>

2. <问题>
   - A. <推荐方案>（推荐）：<影响>
   - B. <备选方案>：<影响>
   - C. <备选方案>：<影响>

若支持点击，请直接点选；若不支持，请直接回复：
- `都按推荐`
- `A / B`
- `第二题改 B`
```

## 产品 PRD 模板入口

正式 PRD 的唯一正文模板维护在 `references/product-prd-template.md`。

在本文件中只保留入口说明，避免与独立 PRD 模板发生双份维护和结构漂移。

使用方式：

- 需要正式 PRD 正文结构时，直接读取 `references/product-prd-template.md`
- 需要“完整交付结构”时，继续参考本文件中的“最终交付结构模板”
- 不要在两个文件中分别维护两份 PRD 正文模板

## 最终检查清单模板

```markdown
## 最终检查清单

| 检查项 | 结果 | 说明 |
|---|---|---|
| 交付模式与章节匹配 | 通过/待补/失败 |  |
| `remainingDebt` 不会改变关键字段/权限/状态/页面/验收，或已显式标为待确认 | 通过/待补/失败 |  |
| `repoConflicts` 已确认或不存在 | 通过/待补/失败 |  |
| `product-prd` 与 `generation-pack` 未串写 | 通过/待补/失败 |  |
| 下游生成门禁状态合法 | 通过/待补/失败 |  |

### 门禁结果
- `continue-confirmation`：仍需继续确认，不可收口
- `ready-for-review`：可输出待确认版本，但不可直接进入下游生成
- `assumption-authorized`：用户已授权在当前假设下继续
- `confirmed`：仅在所有高影响门禁均通过时使用
```
