# 体验问题示例库

这份文件用于快速判断“这是不是问题、通常该怎么写、改进方向大概是什么”。

使用原则：

- 每条示例只提供判断参考，不要求逐字照抄
- 正式报告输出时，问题标题仍必须使用 `序号. [P级]标题` 格式
- 若证据不足，不要因为示例存在就强行下结论

---

## URL 走查示例

### [P0] 主操作不可见

**位置**：页面右上角 / 顶部操作区  
**问题**：核心操作被收进“更多”菜单，首屏不可见。  
**为什么算问题**：高频任务入口被隐藏，会直接影响发现率和操作效率。  
**改进方向**：把核心操作提升为首屏可见的主按钮，放在标题或筛选区附近。

### [P1] 移动端横向滚动无提示

**位置**：移动端列表页 / 表格区  
**问题**：表格溢出后只能横向滚动，但页面没有任何提示。  
**为什么算问题**：用户容易误以为右侧没有内容，造成信息缺失。  
**改进方向**：减少移动端首屏列数，或增加明显的横向滚动提示。

## 截图走查示例

### [P1] 信息层级混乱

**位置**：页面顶部标题区  
**问题**：标题和副标题字号、颜色、字重几乎一致。  
**为什么算问题**：用户无法快速区分页面主题和辅助信息。  
**改进方向**：拉开标题与副标题的字号、字重和颜色层级。

### [P0] 敏感信息未脱敏

**位置**：用户信息区 / 联系信息区  
**问题**：手机号、证件号、邮箱等敏感信息直接完整展示。  
**为什么算问题**：存在明显隐私和合规风险。  
**改进方向**：默认脱敏展示，查看完整信息时增加权限控制或二次确认。

## 流程问题示例

### [P0] 导航层级过深

**位置**：侧边导航 / 菜单结构  
**问题**：菜单层级超过 3 级，进入目标页需要多次展开。  
**为什么算问题**：路径过长会显著增加定位成本，也容易迷失返回路径。  
**改进方向**：压缩层级，把低频节点合并，必要时提供搜索直达。

### [P1] 表单无分组

**位置**：长表单 / 录入页  
**问题**：10 个以上字段连续平铺，没有任何业务分组。  
**为什么算问题**：一次性输入信息过多，用户更容易漏填或放弃。  
**改进方向**：按业务逻辑拆成 2 到 4 组，让每组信息更聚焦。

### [P0] 删除后无法恢复

**位置**：批量删除 / 危险操作区  
**问题**：删除后立即生效，没有确认、撤销或回收入口。  
**为什么算问题**：误操作成本高，且恢复路径缺失。  
**改进方向**：补充二次确认，并提供撤销入口或回收机制。

### [P1] 操作步骤冗余

**位置**：高频编辑流程  
**问题**：一个简单动作被拆成过多步骤，中间还包含重复确认。  
**为什么算问题**：高频流程越长，完成效率越低，也更容易中途退出。  
**改进方向**：合并重复确认步骤，优先缩短高频任务路径。

## 交互问题示例

### [P2] 弹窗关闭方式单一

**位置**：弹窗 / 对话框  
**问题**：只能点击右上角关闭，不支持 Esc 或点击蒙层。  
**为什么算问题**：会增加关闭成本，也不符合多数用户习惯。  
**改进方向**：补齐常见关闭方式，并保持同类弹窗行为一致。

### [P1] 禁用按钮无解释

**位置**：按钮 / 提交操作区  
**问题**：按钮不可点击，但没有任何原因说明。  
**为什么算问题**：用户无法判断是缺少输入、权限不足，还是系统异常。  
**改进方向**：在禁用态附近补充原因提示，或在悬停时解释限制条件。

### [P1] 错误提示过于空泛

**位置**：提交结果提示 / 表单错误提示  
**问题**：页面只提示“操作失败”，不说明原因和下一步。  
**为什么算问题**：用户无法自助修复，只能重复尝试或放弃。  
**改进方向**：提示里直接写清错误原因、影响范围和下一步建议。

### [P1] 加载反馈缺失

**位置**：提交按钮 / 异步操作区  
**问题**：点击后没有 loading、进度或状态变化提示。  
**为什么算问题**：用户容易怀疑是否点击成功，进而重复触发。  
**改进方向**：在按钮或页面层级增加明确的加载反馈，并避免重复提交。

### [P2] 空状态缺失

**位置**：列表页 / 搜索结果区  
**问题**：无数据时直接空白，没有任何解释。  
**为什么算问题**：用户不知道是没结果、没权限，还是加载失败。  
**改进方向**：补充空状态说明，并给出继续操作的建议。

## 样式问题示例

### [P1] 色彩语义不一致

**位置**：全局色彩系统 / 标签 / 状态色  
**问题**：同一种颜色在不同页面代表不同含义。  
**为什么算问题**：会破坏用户对状态的快速判断。  
**改进方向**：统一功能色语义，让颜色和状态一一对应。

### [P2] 数字未使用等宽字体

**位置**：数据表格 / 统计卡片  
**问题**：数字列宽度不一致，纵向难以比较。  
**为什么算问题**：影响高频数据浏览和快速判断。  
**改进方向**：在关键数字区域使用等宽数字能力，提升可读性。

### [P2] 键盘不可操作

**位置**：自定义下拉 / 日期选择 / 树形选择  
**问题**：组件只能用鼠标操作，无法用 Tab、Enter、Esc、方向键。  
**为什么算问题**：会影响无障碍，也降低熟练用户操作效率。  
**改进方向**：补齐基础键盘交互，并保证焦点状态清晰可见。

### [P1] 表格列数过多

**位置**：数据列表 / 订单表格  
**问题**：首屏同时展示过多列，信息拥挤且需要频繁横向滚动。  
**为什么算问题**：用户难以快速抓住关键字段。  
**改进方向**：只保留核心列，低频信息收入展开层或详情面板。

### [P2] 间距无规律

**位置**：卡片布局 / 页面容器 / 表单区  
**问题**：相邻元素间距混乱，没有统一节奏。  
**为什么算问题**：页面会显得不稳定、不专业。  
**改进方向**：统一间距体系，减少临时值和随意调整。

### [P2] 图片显示异常

**位置**：图片容器 / 头像 / 封面图  
**问题**：图片被拉伸、模糊或加载失败后直接裂图。  
**为什么算问题**：会直接破坏页面完整性和专业感。  
**改进方向**：固定容器比例，规范图片适配，并补充失败兜底。

### [P2] Logo 使用不规范

**位置**：品牌 Logo 区域  
**问题**：Logo 被改色、拉伸、旋转，或放在难以识别的背景上。  
**为什么算问题**：会削弱品牌一致性，也影响页面可信度。  
**改进方向**：统一使用标准品牌资源，并保留必要安全空间。
