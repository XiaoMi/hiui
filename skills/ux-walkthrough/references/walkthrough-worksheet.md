# 走查工作表（阶段 B 内部用）

> **用途：** 阶段 B 按批逐项检查，填完再写用户报告与 `report.json`。
> **不写入**对话报告正文或 docx 主文；结论同步到 `checklist_coverage`。

## 使用规则

1. 必须按 **Batch 1 -> 4** 顺序执行，不得跳批
2. 每批填完后自检：本批 ID 是否都已有一行判定？
3. `checklist_coverage` 取值：
   - `pass: <你怎么验证的，>=8 字>` - 已检查且无问题
   - `issue` - 有问题，须在 `issues[]` 体现
   - `pending: <缺什么证据>` - 须写“待交互验证” issue
   - `n/a: <为何不适用>` - 仅当该条对当前页面/模式确实无检查对象
4. **禁止**无证据的裸 `pass`；**禁止**未读 checklist 条目就整批标 pass

## Batch 1 · 路径与可见性（UX-1.1 - UX-1.4）

| ID | 名称 | 判定 | 验证证据（必填） |
|----|------|------|------------------|
| UX-1.1 | 信息架构与任务路径 | | |
| UX-1.2 | 内容展示合理 | | |
| UX-1.3 | 界面一致性 | | |
| UX-1.4 | 功能及信息可见性 | | |

**本批动作：** 枚举可交互面（按钮、列表项、输入框、Tab、hover 操作等）；确认首屏能否看出页面用途与主路径。

## Batch 2 · 反馈与防错（UX-1.5 - UX-1.7, UX-2.1 - UX-2.2）

| ID | 名称 | 判定 | 验证证据（必填） |
|----|------|------|------------------|
| UX-1.5 | 系统状态可见性 | | |
| UX-1.6 | 操作反馈必要性 | | |
| UX-1.7 | 数值高效展示 | | |
| UX-2.1 | 易理解性 | | |
| UX-2.2 | 防错与容错 | | |

**本批动作：** 至少触发 1 次异步/提交类操作，观察 loading 与结果反馈；检查危险操作是否有确认。

## Batch 3 · 一致与可预测（UX-2.3 - UX-2.5）

| ID | 名称 | 判定 | 验证证据（必填） |
|----|------|------|------------------|
| UX-2.3 | 诚实性 | | |
| UX-2.4 | 易学性 | | |
| UX-2.5 | 一致与可预测性 | | |

**本批动作：** 检查弹窗关闭方式（Esc/蒙层）、图标语义、跨区域术语与主操作位置是否一致。

## Batch 4 · 体感与安全（UX-3.1 - UX-3.5）

| ID | 名称 | 判定 | 验证证据（必填） |
|----|------|------|------------------|
| UX-3.1 | 审美性 | | |
| UX-3.2 | 安全性 | | |
| UX-3.3 | 帮助性 | | |
| UX-3.4 | 人性化/情感化 | | |
| UX-3.5 | 用户多样性 | | |

**本批动作：** 检查间距/图标风格；敏感信息是否脱敏；空状态与错误文案是否可读；窄屏/键盘路径是否需 pending。
