# 标注图样式

若本次走查需要生成问题标注图，统一使用“浅红高亮区 + 说明条 + 箭头或短引导线”的样式，不要使用粗红框或大面积实心遮挡块。

## 目标

标注图的目标不是“把问题圈出来”，而是用尽量轻的方式，让读者一眼看懂：

- 问题发生在哪里
- 为什么这里有问题
- 这个问题影响什么

## 推荐生成方式

优先使用脚本自动生成标注图，而不是每次手工临时处理。

推荐命令：

```bash
python3 {SKILL_DIR}/scripts/annotate.py \
  --input raw.png \
  --output issue-1.png \
  --box 120,80,260,96 \
  --label "主按钮文案不清晰，用户难以判断是否会直接提交" \
  --json
```

若当前环境已经通过浏览器工具拿到截图，建议流程如下：

1. 先用浏览器工具完成截图
2. 再用 `annotate.py` 生成标注图
3. 最后把原图和标注图一起用于报告与 `.docx`

若脚本暂时不可用，再退回人工标注；但最终视觉风格仍需遵循本文件规范。

若脚本返回字体告警，说明当前环境没有找到可用中文字体。此时应优先补充字体文件，或通过 `UXW_FONT_PATH` 指向可用中文字体后再生成最终标注图。

## 样式规则

- 问题区域使用浅红半透明高亮，保留原页面内容可读性，不遮挡文字和结构
- 高亮区应覆盖最小必要范围。优先框“一个问题对应的一整块区域”，不要零碎框太多小点
- 问题说明条使用红色文字，一条说明只表达一个问题
- 说明文案应直接描述用户会遇到的问题，不要只写抽象标签
- 说明条与问题区域原则上应有箭头或短引导线连接；若说明条已紧邻问题区域且不会造成歧义，可不强制补箭头
- 箭头或引导线终点应落在问题区域边缘或核心位置，不要插入正文文字中间
- 一张标注图尽量只表达一个核心问题；只有当多个问题属于同一区域、且拆开会增加理解成本时，才允许放在同一张图中
- 标注整体应克制、统一，优先像“点评说明”，不要像“重涂页面”

## 建议参数

- 高亮区：浅红半透明，透明度建议 15% 到 25%
- 说明条：红色文字，浅红半透明背景，圆角 4px
- 箭头或引导线：细线、短路径、避免绕行
- 说明文案：尽量控制为一眼能读完的一句话，优先单行，必要时最多两行

## 禁止项

- 不要默认使用粗实线大红框把内容围死
- 不要使用大面积实心红块覆盖页面正文
- 不要为了编号而默认加角标、小数字标签，除非用户明确要求多点编号说明
- 不要同时混用多套颜色体系，整张图统一使用同一套红色标注语言
- 不要让说明条过长，尽量控制为一眼能读完的一句话

## 标注定位补充规则

### 1. 先找问题本体，再画框

标注前，先明确这条问题真正指向的对象是什么。

- 若问题指向单个按钮、图标、输入框、页签等控件，应优先框该控件本身
- 若问题指向空状态、禁用状态、筛选区等状态块，可框对应状态块
- 禁止先凭整页感觉画大框，再让说明去迁就框的位置

### 2. 优先框控件，不默认框区域

高亮区遵循“最小必要范围”：

- 能框单个控件，就不要框整行
- 能框一组紧邻控件，就不要框整个模块
- 不要因为问题发生在某个模块里，就把整个模块都框起来

判断标准：
读者只看高亮区，不看说明，也应大致知道到底是哪一个对象有问题。

### 3. 多个并列控件有问题时，用多框，不用一个大框

若一个问题对应多个并列控件，应分别框选这些控件，不要用一个大框把它们和中间空白一起包住。

### 4. 小目标先局部确认，再回原图标注

对于宽页面、大截图、右上角入口、表格末列按钮等容易错位的小目标：

- 先从原图裁出局部图确认问题对象
- 确认后再回到原图做最终标注
- 不要直接在整张图上凭肉眼估位置

### 5. 说明文案、箭头和高亮对象必须一一对应

说明里提到什么，就必须真实框到什么。

- 文案写按钮，就必须框到按钮本身
- 文案写输入框，就必须框到输入框本身
- 箭头或引导线终点必须落在高亮区边缘或核心对象上

若说明描述的是 A，但高亮区主要覆盖的是 B，视为标注失败。

### 6. 交付前必须逐张复核

每张标注图在交付前，至少确认以下几点：

- 框到的是不是问题本体
- 有没有把大量无关区域一起框进去
- 说明文字说的对象，是否真的被框到了
- 箭头是否落在正确对象上
- 读者不看上下文，仅看这张图能否理解问题位置

若任一项不满足，不应交付。

## 判断标准

完成标注后，至少自检以下几点：

- 原页面核心内容是否仍然清晰可读
- 读者是否能先看到说明，再顺着箭头或引导线理解问题位置
- 标注本身是否比页面内容更抢眼；若是，则说明标注过重，需要减弱

## 标注前自检

生成最终标注图前，至少检查以下几点：

- 说明提到的对象，是否被高亮区真实覆盖
- 箭头或引导线终点是否落在高亮区上
- 高亮区是否过大，覆盖了太多无关内容
- 说明与标注位置是否一致；若不一致，必须重画
