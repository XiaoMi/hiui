# UX 走查清单

> 来源：HiUI 设计规范 · UX 体验规则
> 用途：逐项检查页面/功能的用户体验质量，记录不通过的项目并给出判断依据。

---

## 一、操作高效

核心目标：用户能快速、高效地完成任务。

### 1.1 信息架构与任务路径清晰

- **检查项**：用户进入页面后，首屏应知道"这是做什么的"和"第一步做什么"
- **判断标准**：首屏是否包含页面标题、主操作入口、核心筛选/搜索
- **严重程度**：P0
- **✅ 推荐**：进入"客户列表"后，首屏能看到页面标题、搜索筛选和"新增客户"按钮
- **❌ 不建议**：首屏堆满统计卡、公告和辅助信息，主操作按钮隐蔽、不在页面上

### 1.2 内容展示合理且完整清晰

- **检查项**：视觉清晰、密度合理、文字易读、重点突出、精简冗余
- **判断标准**：高频信息优先展示，低频信息适当收纳；标题与正文有主次区分
- **严重程度**：P1
- **✅ 推荐**：标题、次要信息在大小、颜色上有主次划分；列表优先展示关键字段，低频字段放入列设置
- **❌ 不建议**：标题与正文颜色样式一样、文字混合排列；大量低频字段占据首屏，关键信息被埋没

### 1.3 界面与交互一致性

- **检查项**：同一系统的内容结构、详情打开方式、返回路径应尽量一致
- **判断标准**：同类型对象在不同模块中的交互模式是否统一
- **严重程度**：P1
- **✅ 推荐**：订单、客户、合同详情都采用相近结构
- **❌ 不建议**：有的开新页、有的开抽屉、有的开弹窗，同类对象交互方式完全不同

### 1.4 功能及信息可见性

- **检查项**：功能易发现、可操作、可预见，文案统一
- **判断标准**：是否有提示标识、状态是否可感知、同功能文案是否一致
- **严重程度**：P1
- **✅ 推荐**：智能填写加 TAG 提示；状态图标加编辑标识；功能文案一致
- **❌ 不建议**：无提示导致信息对应混乱；状态不可感知；同功能文案不一致

### 1.5 系统状态可见性

- **检查项**：操作情境、运行状态、进度是否明确
- **判断标准**：加载/上传/提交等异步操作是否有状态提示
- **严重程度**：P1
- **✅ 推荐**：加载显示"加载中"；上传显示"3/4 剩余 24s"
- **❌ 不建议**：加载仅显示图标无文字，用户无法判断进度

### 1.6 操作反馈必要性

- **检查项**：操作后是否有即时反馈，反馈是否区分结果，是否打断流程
- **判断标准**：提交/保存/删除等操作后是否有明确的结果提示
- **严重程度**：P1
- **✅ 推荐**：提交和保存表单后有 Message 提示"已提交"反馈
- **❌ 不建议**：提交和保存表单后无任何反馈，用户不确定是否成功

### 1.7 数值高效展示

- **检查项**：金额、数量、比例、分值等数值列是否右对齐
- **判断标准**：表格中的数值列是否统一右对齐，便于纵向比较
- **严重程度**：P2
- **✅ 推荐**：订单金额、回款金额、库存数量在列表中统一右对齐
- **❌ 不建议**：数值列左对齐或居中，用户难以纵向比较大小

## 二、符合用户直觉

核心目标：界面符合用户的心理模型和使用习惯。

### 2.1 易理解性

- **检查项**：文案是否面向业务而非系统实现
- **判断标准**：段名、按钮文案、提示文案是否使用业务语言
- **严重程度**：P1
- **✅ 推荐**：提示信息为"库存同步失败，请稍后重试"
- **❌ 不建议**：提示信息为"接口调用异常"（面向系统的错误码）

### 2.2 防错与容错

- **检查项**：是否预防错误、支持撤销、提供清晰指引和友好提示
- **判断标准**：危险操作（删除、停用）是否有二次确认；错误时是否告知原因与解法
- **严重程度**：P1
- **✅ 推荐**：删除、停用等危险操作有二次确认机制；错误提示告知原因和解决方法
- **❌ 不建议**：删除等危险操作无防错机制，可直接删除数据

### 2.3 诚实性

- **检查项**：是否透明、准确、不误导
- **判断标准**：是否如实展示操作/状态/结果，有无隐藏关键信息或虚假提示
- **严重程度**：P2
- **✅ 推荐**：如实展示操作状态和结果
- **❌ 不建议**：隐藏关键信息、虚假提示（如显示"成功"实际未成功）

### 2.4 易学性

- **检查项**：是否易学习、复用习惯、认知负担低
- **判断标准**：是否沿用通用操作逻辑，还是不断引入复杂新概念
- **严重程度**：P2
- **✅ 推荐**：沿用通用操作逻辑（如双击编辑、右键菜单）
- **❌ 不建议**：不断新增复杂概念，用户需要反复学习

### 2.5 一致与可预测性

- **检查项**：结果是否符合预期、变化是否可预判
- **判断标准**：同类对象是否使用同类结构；相同行为是否使用相同模式；术语和状态语义是否全局统一
- **严重程度**：P1
- **✅ 推荐**：所有列表页都把"新增"放右上角，"导出"放在工具栏次级位置
- **❌ 不建议**：A 页面保存在顶部，B 页面在底部，C 页面在浮层里，位置不统一

## 三、好的体感

核心目标：界面美观、安全、有帮助、有人情味。

### 3.1 审美性

- **检查项**：美观服务于功能，视觉是否统一
- **判断标准**：图标风格、色彩运用、视觉层次是否协调一致
- **严重程度**：P2
- **✅ 推荐**：Icon 统一灰色，聚焦文字内容
- **❌ 不建议**：Icon 色彩杂乱、视觉分散，干扰信息获取

### 3.2 安全性

- **检查项**：是否保护身份信息、企业数据、合规操作
- **判断标准**：账号信息是否有安全防护，敏感数据是否有加密/脱敏
- **严重程度**：P0
- **✅ 推荐**：密码输入有加密显示功能，敏感信息脱敏展示
- **❌ 不建议**：信息泄露、不合规流程（如明文展示密码）

### 3.3 帮助性

- **检查项**：是否主动帮助、场景化提示
- **判断标准**：是否有智能引导、主动提示操作，还是被动求助、无指引
- **严重程度**：P2
- **✅ 推荐**：智能引导、主动提示操作（如首次使用弹出引导气泡）
- **❌ 不建议**：被动求助、无指引、无提示，用户只能自己摸索

### 3.4 人性化/情感化

- **检查项**：是否有人性化表达和情感沟通
- **判断标准**：语气是否友好、不机械，是否有温度
- **严重程度**：P2
- **✅ 推荐**：语气友好、不机械（如"已经帮你保存好了"而非"保存成功"）
- **❌ 不建议**：语气冰冷生硬、机器化沟通

### 3.5 用户多样性

- **检查项**：是否适配不同设备/场景/习惯，兼顾差异
- **判断标准**：是否兼容多端、适配不同用户群体
- **严重程度**：P2
- **✅ 推荐**：兼容多端、适配不同用户（如大屏/小屏、新手/专家）
- **❌ 不建议**：仅适配单一场景，忽略其他终端或用户群体
