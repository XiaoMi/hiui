# 体验走查报告模板

按此结构输出最终报告。占位符用 `{...}` 表示。

除在对话中输出 Markdown 报告外，还必须同步生成一份本地 `.docx` 报告。
本地 `.docx` 报告的生成要求见 `report-docx.md`。
结构化 `report.json` 的字段约定见 `report-json.md`。
截图证据是否为必需、以及不同模式下的差异要求，以对应 mode 文档为准。

## 正文写法原则

- 语言通俗，不堆术语
- 每个问题只写最小必要信息：位置、问题描述、改进建议
- 同类问题合并，避免重复报
- 问题标题固定使用 `序号. [P级]标题` 格式，例如 `1. [P1] 主操作入口不明显`
- 问题详情中不要输出“对应原则”字段
- 代码走查没有截图时，不要强行补“截图”字段

## 对话展示要求

- 对话中直接输出完整 Markdown 报告，供用户即时预览
- 不要只输出摘要、结论或文档链接
- 对话中若有截图，优先直接展示可预览图片
- 若无法直接展示图片，至少提供可点击预览的图片方式
- 不要只保留图片路径或文件名而不让用户直观看到图片
- 在完整报告结束后，附上本地 `.docx` 文件路径
