# 忽略问题清单

以下场景在 B 端走查中应**忽略不报**或**降级为待确认**，避免误判。

> 与 `ux-checklist.md` 中 `ignore_boundary` 字段配合使用。

---

## 1. 信息密度过高 / 屏效设计

**忽略原因**：B 端强调屏效，一屏展示更多数据常为设计意图。

**包含但不限于**：

- 表格行间距小、单屏行数多
- 搜索 placeholder 多字段（如「搜索名称/编号/手机号」）
- 看板/仪表盘数据密集
- 侧边栏、面包屑、标签页占用空间（必要导航结构）

**与 checklist 边界**：

| checklist | 仍应忽略 | 仍应上报 |
|-----------|----------|----------|
| UX-1.1 | 首屏有标题+主 CTA+搜索，统计卡多 | 主 CTA 被挤没、折叠进「更多」 |
| UX-1.2 | 行间距小但关键列可读 | 列数 >9 且无列设置；关键信息被埋没 |
| UX-1.7 | — | 数值对齐/等宽问题仍应报 |

**判断句**：密度服务于「少翻页、少跳转」→ 忽略；密度导致「找不到主路径、读不懂主次」→ 上报。

---

## 2. 代码走查无法静态证明的项

**忽略原因**：仅读代码/单张截图，无法可靠验证运行时行为。

**默认标「待交互验证」，不写确定结论**：

- 悬停、键盘、拖拽等交互细节（除非代码明确缺失）
- 动画时长、加载耗时
- 真实接口错误文案（除非 hardcode 可见）

**关联 checklist**：UX-1.5、UX-1.6、UX-2.4、UX-3.5 在 `source=code` 且无运行时证据时，优先待确认。

---

## 3. 主观性强 / 证据不足项

**忽略原因**：缺少可复现证据时，易变成风格偏好。

**默认不报，除非有明确反例**：

- UX-3.4 情感化：文案「不够温暖」类主观评价
- UX-3.5 多端：仅一张桌面截图却报「移动端适配差」
- UX-3.3 帮助性：未走完整新用户路径却要求「必须有引导」

**例外**：有截图/录屏/多 viewport 证据时，仍正常上报。

---

## 4. 品牌内部规范（非通用 UX）

**忽略原因**：走查聚焦可用性，品牌细则另审。

- Logo 像素级规范（除非明显违规且项目声明遵循品牌手册）→ 见 UX-3.1 示例，有明确违规仍报
- 内部 Design Token 命名是否统一 → 不报，除非导致用户可见混乱（UX-2.5）

---

## 5. HiUI 典型页模板默认（basic-table）

**忽略原因**：页面按 HiUI 官方 **基础表格** 典型页实现时，下列为 **模板合规默认**，不是 UX 缺陷。

**前置条件**：已识别 HiUI 上下文且页型为 `basic-table`（识别规则见 `hiui-template-baseline.md`）。

**包含但不限于**（细节与偏离例外见 baseline 全文）：

| 场景 | 处理 |
|------|------|
| 表格 **10 列**（含操作列） | 忽略 UX-1.2「列过多」 |
| 行内 **查看 + 编辑** 双 link | 忽略「操作过多 / 主次不清」 |
| **1 个**合并 SearchInput（多字段 placeholder） | 忽略 §1「搜索 placeholder 多字段」 |
| Mock **手机号/邮箱明文** | 忽略 UX-3.2；**生产/合规场景例外** → 仍报 `[业务域]` |
| 分页 mock **共 100 条 / 20 条/页** | 忽略「演示数据量」类问题 |
| 手机号/工号等 **左对齐**（非 tabular-nums） | 忽略 UX-1.7 |

**仍应上报**（相对模板偏离，不可 ignore）：

- 筛选维度与表格列 **不对齐**（有筛选项、无对应列）
- 页头 title 与菜单 route title **不一致**
- 空表仍展示分页、筛选重置无效等（见 baseline §偏离模式）

**与 checklist 边界**：

| checklist | HiUI basic-table 仍忽略 | 仍应上报 |
|-----------|-------------------------|----------|
| UX-1.2 | 10 列高密度 | D1 筛选列不对齐；关键列被删且任务受阻 |
| UX-1.7 | 模板默认左对齐文本列 | — |
| UX-3.2 | mock 明文 | 用户声明生产/HR/财务合规场景 |

### full-page-edit（全页编辑）

**前置条件**：已识别 HiUI 上下文且页型为 `full-page-edit`（见 `hiui-template-baseline.md` §full-page-edit）。

| 场景 | 处理 |
|------|------|
| 表单 **三栏**、字段 **多**、需 **区内滚动** | 忽略 UX-1.2「信息过密 / 一屏放不下」 |
| 标签在 **控件上方**（top） | 忽略「标签占空间」 |
| 底栏 **取消 + 暂存 + 提交** 靠右固定 | 忽略 UX-1.4「按钮过多」、UX-2.5「底栏位置不统一」 |
| 底栏 **无阴影**、仅顶部分割线 | 忽略「底栏不够醒目」类主观项 |
| 分组 **Card** 多块（如基础/客户/服务信息） | 忽略「区块太多」 |
| Mock 表单内 **电话明文** | 忽略 UX-3.2；**生产/合规例外** → 仍报 `[业务域]` |

**仍应上报**：>16 字段用 Drawer（ED1）、无 onBack（ED2）、侧标 label（ED3）、整页滚走底栏（ED5）、横向滚动（ED6）等，见 baseline §偏离模式。

### full-page-detail（全页详情）

**前置**：页型 `full-page-detail`。

| 场景 | 处理 |
|------|------|
| 多块分组、**3 列**描述、信息密度高 | 忽略 UX-1.2「字段太多/过密」 |
| 分组 **无描边** 白底块、组间 16px | 忽略「区块弱区分」 |
| 长文本 **通栏 colSpan** | 合规默认 |
| Mock 明文 | 忽略 UX-3.2；生产/合规 → `[业务域]` |

**仍报**：>16 仍抽屉（FD1）、全页误用 2 列·16（FD3）、无 onBack（FD2）等。

### drawer-form（抽屉表单）

**前置**：页型 `drawer-form` 且 **≤16 字段**。

| 场景 | 处理 |
|------|------|
| **360/600 分档宽度**、双列 **gutter 32** | 忽略「抽屉窄/宽不对」（若符合分档表） |
| **`rowGap: 0`**、标签 **top** | 合规 |
| **无 Card 分组**、单分组 **无组标题** | 合规 |
| 取消+确认 **靠右**、footer **无顶线** | 合规 |

**仍报**：>16 仍抽屉（WD1）、rowGap 未 0（WD2）、Card 分组（WD4）等。

### drawer-detail（抽屉详情）

**前置**：页型 `drawer-detail` 且 **≤16 项**。

| 场景 | 处理 |
|------|------|
| **360/600 分档**、**2 列·16**（非全页 3·40） | 合规 |
| **无 Card**、标题 **16px** | 合规 |
| 列表嵌套 **无遮罩** | 合规（典型） |
| Mock 明文 | 同 B7 |

**仍报**：>16 仍抽屉（VD1）、3 列·40（VD2）、Card 外包（VD3）、标签未左对齐（VD4）等。

### table-stat / data-visualization（统计·可视化）

**前置**：页型 `table-stat` 或 `data-visualization`。

| 场景 | 处理 |
|------|------|
| 顶部 **指标卡/多图表**、信息密度高 | 忽略 UX-1.2 |
| 统计页筛选 **16/20** | 合规（非普通列表 padding） |
| 默认 **无同环比**、表格 **多列指标** | 合规 |
| 合并搜索 + 全部筛选 | 同 basic-table |

**仍报**：白卡未铺满露灰（SD1）、表体不可滚（SD2）、stickyTop 误配（SD3）等。

### tree-table（树形表格）

| 场景 | 处理 |
|------|------|
| 树形首列较宽、**编辑+子部门** 双 link | 合规 |
| 根节点少时分页 **仍显示** | 合规（勿误报缺分页） |

**仍报**：子行名称空白/错行（TD1）、`autoHide: true`（TD2）。

### tree-split（左树右表）

| 场景 | 处理 |
|------|------|
| **左树 + 右表** 双搜索框 | 合规 |
| 树选中刷新表、与右筛无默认联动 | 合规 |
| 右表筛选 **20/20/16/20** | 合规 |

**仍报**：误用 stat 筛选/指标卡（RD1）、双表头错位（RD3）。

### exception-page（异常页）

| 场景 | 处理 |
|------|------|
| 单焦点 EmptyState + 简短文案 + 居中按钮 | 合规 |
| **场景化 SVG 插图** | 合规（非表格灰图规则） |
| 页头演示用 extra 按钮 | 忽略 UX-1.4 |

**仍报**：无主路径 CTA（XD1）、异常类型与文案不符（XD2）。

---

## 6. 持续补充

新增 ignore 规则时，请同步更新对应 checklist 条目的 `ignore_boundary` 字段及 `hiui-template-baseline.md`（若涉及 HiUI 页型）。
