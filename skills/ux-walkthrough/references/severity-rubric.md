# 优先级判断表

## P0

满足任一条件即可判定为 P0：

- 阻断主流程，用户无法完成核心任务
- 错误后几乎无法恢复，且影响范围大
- 关键入口不可见或关键结果不可达

## P1

满足任一条件即可判定为 P1：

- 不阻断任务，但明显拖慢效率
- 用户容易困惑、误解、重复操作
- 缺少必要反馈、引导、解释、恢复路径
- 在高频场景中持续造成摩擦

## P2

满足任一条件即可判定为 P2：

- 不影响完成任务，但影响专业感、一致性、可读性
- 样式、文案、图像、品牌使用不规范
- 低频场景下的轻度体验问题
- 需要优化，但不应抢占更高优先级问题

## 深度问题定级补充

### 可判 P0

- 核心任务链路断裂，用户无法完成主要业务目标
- 状态流转缺失导致任务无法继续，例如驳回后无法修改重提
- 权限错配导致越权访问、敏感数据泄露，或关键角色无法完成职责
- 页面反馈与真实结果相反，例如显示成功但实际失败
- 异常后数据丢失且无法恢复，例如提交失败后清空已填写内容
- 批量、审批、财务、权限等高风险操作无确认且不可恢复
- 创建、提交、发布等核心对象闭环完全断裂，例如无法创建成功、创建后结果不可达、详情不可进入且无替代路径

### 可判 P1

- 不完全阻断任务，但明显增加重复录入、绕路、人工确认或沟通成本
- 某些业务状态缺少对应操作、解释或下一步，但仍有替代路径
- 无权限、禁用、失败、空态等状态缺少原因和解决路径
- 表单字段联动、默认值、必填规则不符合业务预期，导致用户反复试错
- 跨页面入口、返回、编辑、保存后的落点不一致，持续造成困惑
- 批量操作、导入导出、审批处理等复杂任务缺少结果明细或失败明细
- 上下文操作的图标/文案语义与真实动作不一致，用户容易误解为删除、关闭、移除或取消高价值对象
- 高频对象上的可点击、滑动、拖拽、长按、hover 快捷操作缺少可发现线索或反馈，导致用户不知道功能存在或误判为不可用
- 创建/编辑/删除/邀请等任务只验证到入口或弹窗，未能验证提交结果、列表刷新、详情可达、状态变化或回收路径
- 创建成功后对象不出现在预期列表，或需要手动刷新、重新搜索、切换页面才能看到结果
- 删除、撤销、归档后列表、详情、计数或状态没有同步更新，用户需要额外确认真实结果

### 仍判 P2

- 不影响任务完成，只影响可读性、专业感或局部一致性
- 深层风险没有足够证据，只能作为待确认项
- 低频状态的轻微提示不完整，但不影响恢复
- 样式、文案、布局问题未影响理解和操作

## 降级规则

- 仅视觉不统一、但不影响理解和操作：优先判 P2
- 仅信息密度高、但提升效率：先参考 `ignore-list.md`
- 证据不足时，不直接升到 P0/P1，优先标记为待确认
- 证据不足的深度问题，不直接判 P0/P1；除非已有代码、截图或交互证据能证明主流程受阻
