# 优先级判断表

## P0

满足任一条件即可判定为 P0：

- 阻断主流程，用户无法完成核心任务
- 错误后几乎无法恢复，且影响范围大
- 关键入口不可见或关键结果不可达

## P1

满足任一条件即可判定为 P1：

- 不阻断任务，但明显拖慢效率
- 用户容易困惑、误解、重复操作
- 缺少必要反馈、引导、解释、恢复路径
- 在高频场景中持续造成摩擦

## P2

满足任一条件即可判定为 P2：

- 不影响完成任务，但影响专业感、一致性、可读性
- 样式、文案、图像、品牌使用不规范
- 低频场景下的轻度体验问题
- 需要优化，但不应抢占更高优先级问题

## 降级规则

- 仅视觉不统一、但不影响理解和操作：优先判 P2
- 仅信息密度高、但提升效率：先参考 `ignore-list.md`
- 证据不足时，不直接升到 P0/P1，优先标记为待确认
