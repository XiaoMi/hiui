# Legacy-host-compatible Data Visualization Template

This strict template belongs to the `legacy-host-compatible` bridge path. That mode still supports ordinary typical pages through `page-component + runtimeAdapterProof`, but data-visualization remains `managed-analytics`, so this template is a strict fallback/start-point asset rather than proof that legacy cannot use page components.

Strict-example pages in `legacy-host-compatible` mode must start from `page.template.tsx` in this directory. Falling back to a synthetic scaffold is not allowed.
