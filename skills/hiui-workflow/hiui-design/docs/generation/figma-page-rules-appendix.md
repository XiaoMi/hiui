## 何时再看本文件

- 需要处理容易混淆的页型边界
- 需要确认两套相近规则的差异点

如果只是正常生成页面，先只看 `page-type-map.md`、`figma-page-rules.md` 和对应专章即可。

## 易混对照

### 数据统计表 vs 左树右表

- 数据统计表重点是“指标卡 + 筛选 + 表格 + 分页”一体白卡
- 左树右表重点是“左树 + 右表”必须在同一个 split frame 内
- 两者筛选区 `padding` 不同：
  - 数据统计表：`16px 20px`
  - 左树右表右侧：`20px 20px 16px 20px`

### 抽屉表单 / 抽屉详情 vs 全页页型

- 可编辑字段 `<=16`：优先抽屉表单；`>16`：切全页编辑
- 只读描述项 `<=16`：优先抽屉详情；`>16`：切全页详情
- 不要通过“继续加宽抽屉”规避页型切换

### 抽屉详情 vs 全页详情

- 抽屉详情标题默认 `16px / 600`
- 全页详情标题默认沿用全页页头节奏，不要把抽屉标题样式直接套过去
- 抽屉详情双列时长文本独占整行；全页详情默认三列，长文本同样要占满整行

### 全页编辑 vs 通用后台表单页

- 全页编辑必须保留固定页壳的头部、滚动区、底栏关系
- 如果页面看起来像“灰底上的单张大白卡表单”，通常不是业务字段问题，而是已经退回通用后台脚手架
- `inlineEditFooter` 不是可随意替换的视觉实现细节，而是页型的一部分

## 全局失真判断

### 优先判断为样式资源失效的信号

- 多个页型同时出现白卡、圆角、标题区、固定底栏一起失真
- 数据统计页指标卡退化成普通文本块
- 白底壳层整体消失，而不是单个局部 spacing 偏差

### 优先判断为宿主高度链失效的信号

- 多个页型同时出现页头贴边
- 主区白底不铺满到底部
- 抽屉 / 全页 / 表格类页面同时出现双滚动
- 表格高度不收口、分页和表体节奏整体漂移

## 放不进正文、但值得记住的补充

- 左树搜索框需要显式 `style={{ width: '100%', maxWidth: '100%', minWidth: 0 }}`
- Picker 系控件默认 `matchWidth=true`，需要显式关闭
- 不要只在统计页单独改 `.hi-v5-table-footer`
- Figma URL 里的 `node-id=12-345`，在 MCP 里要写成 `12:345`
