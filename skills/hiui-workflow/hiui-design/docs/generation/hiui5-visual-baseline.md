# HiUI 5 典型页视觉与组件基线

## 角色声明

本文件用于承接 HiUI 5 的**视觉气质、颜色层级、字阶和通用组件默认样式**，供典型页生成时复用。

它解决的是：

- 新页面没有逐项设计稿时，典型页仍应保持统一 HiUI 5 视觉基线
- 共享规则里不适合反复展开的颜色、字阶、按钮、输入框、表格默认样式，需要有一个集中参考

它不替代：

- `page-type-map.md` 的页型判断
- `figma-page-rules.md` 的默认 `nodeId` / 页壳注册
- `figma-pages/*.md` 的页型 P0 / 禁止项
- 当前项目 `typical-page-patterns.md` 的代码落地细节

发生冲突时，优先级如下：

1. `figma-pages/*.md`
2. `figma-pages/forms-shared.md` / `figma-pages/table-shared.md`
3. `generation-principles.md`
4. 本文件

## 使用方式

- 命中已有页型时，优先看页型专章；只有需要补视觉与组件默认值时，再回看本文件
- 未命中现成页型时，先读 `generation-principles.md`，再用本文件补足 HiUI 5 基线
- 不要把本文件当成“所有页面都必须逐字照抄的数值表”；页型专章已有明确规定时，以专章为准
- 对 `hiui-design` 受管页面，本文件同时承担视觉 token 硬门槛：页面源码与页面本地样式里直接出现的原始字号、颜色、投影，默认只允许落在本文件登记的基线与页型专章明示例外内
- 对跨语言文案、格式化与 RTL 方向，统一看 `i18n-baseline.md`；本文件只负责与字阶、颜色、按钮、表格外观直接相关的视觉基线

## 外部组件 token bridge

外部项目若必须接入非 HiUI 组件，本文件同样是它的视觉 token 事实源。统一按下面口径处理：

- 先判断是否真的存在 HiUI 能力缺口；若 HiUI 已有语义组件或 approved helper，优先复用，不得为了局部交互偏好直接切回第三方默认实现
- 第三方组件必须先经过 adapter / token bridge，再进入业务页面；不要在页面源码里直接散落该库自带的 theme provider、palette、字号体系或默认 surface
- adapter / token bridge 中出现的原始颜色、字号、圆角、边框、阴影，仍只允许使用本文件登记的基线或页型专章批准的例外
- 第三方组件可以承接局部行为能力，但不能顺手定义页面主色、标题层级、卡片层级、滚动 owner 或第二套白底/灰底关系
- 若库支持 theme object、design token 或 CSS variables，默认应把这些入口映射到 HiUI 基线，而不是沿用库默认 palette / typography / radius / shadow

推荐最小映射至少覆盖：

- 颜色：primary、text-primary、text-secondary、border、surface-primary、surface-muted
- 字号：page title、section title、body、meta、metric-overview、metric-embedded
- 容器：control radius、panel radius、panel border、panel padding

禁止把下面这些值视为“库内部样式所以不受管”：

- 第三方默认蓝紫主色、默认中性色阶、默认 status palette
- 未登记的 `font-size`、`line-height`、`box-shadow`
- 由第三方卡片 / 面板默认样式带出的第二套 surface hierarchy

## 页面气质

- 整体应保持 B 端企业系统的克制感，以信息密度、识别效率和操作稳定性优先，不靠装饰制造差异
- 默认视觉关系是“灰底外槽 + 白底主体内容区”，而不是大面积彩色底或强装饰卡片墙
- 主交互色默认只认 HiUI Blue：`#2660FF`；没有书面设计依据时，不要为新页面发明第二套主色
- 语义色只承担语义，不承担主结构装饰：
  - 危险 / 错误：`#FA4646`
  - 成功：`#24B237`
  - 警告：`#FFBE0A`
  - 信息：`#04C2AC`
- 阴影默认不用；没有书面设计依据时，受管页面禁止新增投影；若现有壳层已经是描边白卡，就不要再叠任何额外投影

## 颜色基线

### 主交互蓝

- 主按钮 / 激活 / 选中 / 链接：`#2660FF`
- hover：`#4D82FF`
- pressed：`#1843D2`
- 浅色交互底：`#EDF2FF`
- pressed 态浅底 / outline pressed wash：`#E5ECFF`

### 中性色

- 主标题 / 主要正文：`#1A1D26`
- 表单标签 / 次一级正文：`#4E5969`
- 次要说明：`#60636B`
- 占位 / 元信息：`#91959E`
- 禁用文字：`#BABCC2`
- 默认边框 / 分隔线：`#E6E8EB`
- 次级表面：常用 `#F2F4F7`；弱强调填充常见 `#EDEFF2`
- 主白底：`#FFFFFF`

### 使用约束

- 典型页默认优先依赖“白底 + 灰底 + 蓝色主交互”三层关系
- 紫色、粉色、靛蓝等强调色不作为典型页主流程结构色；仅在图表、特殊分类或产品明确要求时再使用
- 颜色优先服务语义和层级，不要把颜色当成新的页面分区系统
- 受管页面中直接出现的十六进制颜色，只允许使用本文件已登记的颜色值或 `figma-pages/*.md` 中书面声明的设计特例
- 受管页面禁止自行新增未登记的十六进制颜色，也禁止直接写 `rgb(...)` / `rgba(...)` / `hsl(...)` / `hsla(...)`

### Surface Ownership

- `#FFFFFF` 是一级主体区、一级详情卡、图表卡、时间线主卡、反馈主面板的默认主表面
- `#F2F4F7`、`#EDEFF2` 等浅灰底不是禁用色，但默认只允许用于页面外槽、内部包裹层、弱强调填充或二级信息面
- 若页面已存在稳定的白底主体区，不要再把一级结构卡整体刷成浅灰底制造第二套层级
- 没有书面设计依据时，不要用“浅灰底大卡片 + 描边 + 大面积留白”替代应由白底、字阶和分组节奏表达的信息层级

## 字体与字阶

- 默认 UI 字体优先沿用 HiUI / 项目注入字体栈，不在典型页规则层强指定 `MiSans VF` 这类单一字体
- 若项目未额外注入字体，可回落到 `Helvetica Neue, PingFang SC, Microsoft YaHei, sans-serif`
- UI 层字重默认只用 `400 / 500 / 600`，不要无依据引入 `700`

### 推荐层级

- 页面级标题：`18 / 26 / 600`
- 区块 / 卡片标题：`16 / 24 / 500~600`
- 子分组标题 / 右侧详情主标题：`14 / 22 / 500~600`
- 表头 / 分组小标题：`14 / 22 / 600`
- 正文 / 表格单元格 / 普通标签：`14 / 22 / 400`
- 辅助说明 / Tag / 元数据：`12 / 20 / 400~500`
- 指标主值：`24 / 32 / 600`（`overview metric surface`）或 `16 / 24 / 600`（`embedded metric surface`）

### 使用约束

- 标题应依赖字重和节奏建立层级，不要额外叠很重描边、底纹和装饰块
- 同一页内不要把多个层级都做成 `18px`
- 页面主标题、区块标题、右侧详情标题必须分档；不要把所有标题都做成同一尺寸
- 业务文本容器默认必须接受西文、泰文与阿拉伯语的长度膨胀；不能只按中文文本长度写死窄宽与固定高
- 受管页面源码或页面本地样式里直接出现的原始 `font-size`，文本层级默认只允许 `12px`、`14px`、`16px`、`18px`；指标主值额外允许 `24px`
- 指标主值不是通用字号；只允许使用 `24px / 32 / 600` 或 `16px / 24 / 600`
- `24px / 32 / 600` 只属于 `overview metric surface`；是否属于 overview 默认按阅读任务与是否重复性出现在卡片列表中判断，不按窄侧栏、`secondary` 或左侧卡片列位置单独决定；若该 token 位于受限宽度区域且确认必须保留，仍需满足该 KPI 组的最小宽和列策略，不得再写死等宽三列
- 若同一页面同时存在 overview / embedded 两类指标组，推荐在指标组容器上显式回写 `data-hiui5-metric-surface="overview|embedded"`；`doctor` 会优先读取该 marker，再回退到语义 selector 与弱推断
- 禁止在页面局部自行新增 `13px`，也禁止为了 hero、指标卡、摘要卡、时间线、经营看板或局部信息块再发明 `20px`、`22px`、`28px`、`30px`、`32px`、`40px` 等新字号
- 除 HiUI 组件内部已封装的文本外，业务自定义文本默认必须通过语义类名或同等级的语义样式宿主显式声明字号；不要依赖浏览器默认 `16px`

### 正式角色矩阵

文本角色：

- `18px`：页面级标题、页内实体主标题
- `16px`：一级区块标题、图表卡标题、抽屉标题、弹窗标题
- `14px`：正文、列表项标题、统计卡说明标题、指标名称、表单标签、常规字段标题
- `12px`：元信息、辅助说明、Tag、时间口径、来源说明、补充注释

指标角色：

- `24px / 32 / 600`：`overview metric surface` 主值，例如独立总览指标卡、`stat-section`、通栏摘要带、非重复实体卡片列表中的核心总览 KPI
- `16px / 24 / 600`：`embedded metric surface` 主值，例如重复性卡片列表中的指标、列表卡内部、副栏局部摘要、详情头本地统计

这份矩阵只约束业务自定义文本角色，不追究 HiUI 组件内部由组件封装负责的文字样式。

### 字号语义映射

受管页面里的字号不只要“值合法”，还必须“角色合法”。同一个字号不得在不同层级之间随意漂移。

#### `18px`

- 允许：页面级标题、页内实体主标题、hero 主对象名称
- 禁止：一级区块标题、图表卡标题、抽屉标题、正文、元信息、时间线项标题
- 判断原则：`18px` 回答的是“我正在看谁”，不是“这个区块讲什么”

#### `16px`

- 允许：一级区块标题、图表卡标题、抽屉标题、弹窗标题、一级面板标题；若作为指标主值，必须同时满足 `line-height: 24px` 与 `font-weight: 600`，且场景限定为 `embedded metric surface`
- 禁止：页面主标题、hero 主对象名称、以普通 `16px` 文本样式冒充指标主值、元信息
- 判断原则：`16px` 是结构切片标题，负责表达区块边界

#### `14px`

- 允许：正文、表头、表单标签、常规标题、列表项标题、统计卡说明标题、指标名称
- 禁止：页面主标题、一级区块标题、指标主数字、需要弱化的元信息
- 判断原则：`14px` 是默认信息表达字号；不需要强调和弱化时，优先回到 `14px`

#### `12px`

- 允许：元信息、辅助说明、Tag、区块补充说明、时间与口径信息
- 禁止：页面主标题、一级区块标题、正文主表达、指标名称、统计卡说明标题、指标主数字
- 判断原则：`12px` 只能用于“可读但不抢层级”的内容

#### `24px / 32 / 600`

- 允许：独立总览指标卡、`stat-section`、通栏摘要带、非重复实体卡片列表中的核心总览 KPI 等 `overview metric surface` 主值
- 禁止：重复性卡片列表中的指标、列表卡内部、副栏局部摘要、详情头本地统计、页面标题、区块标题、正文、摘要、标签、一般数字
- 判断原则：`24px / 32 / 600` 只回答“总览级指标主值”；核心判断是阅读任务而非位置，若指标已重复性出现在卡片列表中，默认不再使用该 token

#### `16px / 24 / 600`

- 允许：重复性卡片列表中的指标、列表卡内部、左侧选择卡、副栏局部摘要、详情头本地统计等 `embedded metric surface` 主值
- 禁止：独立总览指标卡、`stat-section`、页面标题、区块标题、正文、标签、一般数字
- 判断原则：`16px / 24 / 600` 与 `24px / 32 / 600` 属于同一语义角色的不同 surface 版本，不是普通正文或标题字号

### 组件优先级与默认选型

- 命中 HiUI 已有语义组件的场景，默认先复用现成组件，再考虑局部样式覆盖
- 时间流 / 审批轨迹 / 行为记录 / 关键动作：优先 `Timeline`
- 空状态：优先 `Empty`
- 状态表达：优先 `Tag`
- 只读字段分组：优先 `Descriptions` / schema descriptions
- 百分比或进度表达：优先 `Progress`
- 步骤流：优先 `Steps`
- 不要在组件已存在、语义匹配、数据可映射的前提下，先手写等价 DOM 再补样式
- 若必须通过非 HiUI 组件补能力，默认仍要让 HiUI 语义组件负责页面主表达；第三方组件只允许留在 adapter / bridge 承接的局部 body 区域

## 按钮基线

- 按钮圆角默认 `6px`
- 高度默认分档：
  - `24px`：超小型行内按钮
  - `28px`：紧凑工具按钮
  - `32px`：默认表单 / 筛选 / 列表操作按钮
  - `40px`：强调级主操作
- 主按钮默认蓝底白字；危险操作默认红底白字；次级操作优先 outline；弱操作优先 text / link
- Link 只用于行内文字动作、跳转和轻交互，不要拿来承担主提交
- 非中文按钮文案过长时，优先缩减为更稳定的动作语义，或切回 icon + 通用词；不要把中文短文案逐字硬翻后继续塞进固定窄按钮

### 使用约束

- 每个操作区默认最多 1 个主按钮
- 危险按钮只用于删除、移除、停用等破坏性确认，不用于普通导航
- 页头 / 表头操作区若同组存在多个按钮，主按钮仍应放在最右

## 卡片与容器基线

- 默认关系：白卡放在灰底区上
- 默认边框：`1px solid #E6E8EB`
- 圆角：
  - 紧凑卡片：`8px`
  - 标准卡片 / 主体分区：`12px`
- 标准内边距默认 `20px`
- 默认不要依赖重阴影；已有描边白卡足够表达区块层级

### 使用约束

- 卡片是内容分组单元，不是额外装饰
- 不要在典型页里无限嵌套卡片；一般不应超过 2 层
- 若现有固定页壳已承担白底主体，不要再在主体内部重复堆“白卡套白卡”
- 受管页面默认只允许平面白卡与描边层级；若没有书面设计依据，不要在页面局部新增 `box-shadow`
- 默认允许的投影值只有 `box-shadow: none`

## 输入框与表单基线

- 默认高度：`32px`
- 小尺寸：`28px`
- 大尺寸：`40px`
- 默认圆角：`6px`
- `appearance="line"` 常见边框：`1px solid #DBDDE0`
- 正文文字：`14px / 400 / #1A1D26`
- 标签文字：`14px / 400 / #4E5969`
- 占位色：`#91959E`
- focus 态：通常以主色边框 `#2660FF` 为主；若具体组件存在额外焦点样式，以组件默认实现为准
- disabled：背景常见 `#F2F4F7`；文字 / 图标通常落在 `#91959E ~ #BABCC2`

### 使用约束

- 典型页内优先沿用 HiUI 5 默认输入控件外观；不要为了“统一感”手写一层新的 filled / ghost 语法
- `QueryFilter` / `FilterDrawer` 的展示逻辑优先交给共享桥接或组件默认值，不要把表现层写死进业务字段
- `QueryFilter` 内的“统一感”应通过共享 `filter surface baseline` 达成，而不是把所有文本字段都直接改造成 `SearchInput`
- 关键词搜索位保持 `SearchInput` / 搜索语义；普通文本筛选字段应使用与其同皮肤、但语义独立的受管筛选输入角色
- 若共享层当前无法让文本 / 选择 / 日期字段落在同一筛选表面，应先补共享层能力或显式记录 waiver；不要让业务页各自手写灰底补丁

## 表格基线

- 表头背景：无额外填充背景（沿用表格白底或透明背景）
- 表头文字：`14px / 500 / #91959E`
- 单元格文字：`14px / 400 / #1A1D26`
- 行分隔线：`1px solid #E6E8EB`
- hover 行底：`#F2F4F7`
- selected 行底：`#EDF2FF`
- 默认密度优先 `size="md"`

### 使用约束

- 典型页表格默认追求稳定、易读，不追求高装饰化 striped 视觉
- 若没有页型专章明确要求，默认不单独改小或改大表格密度
- 表格的颜色差异主要用在 hover、selected、状态 Tag，不要把整张表做成多色块信息墙

## Tag / 状态表达基线

- 默认 Tag：`filled default` 常见为 `#EDEFF2` 底 + `#1A1D26` 字
- 主状态 Tag：浅蓝底 + 蓝字
- 危险状态 Tag：浅红底 + 红字
- 成功状态 Tag：浅绿底 + 绿字

### 使用约束

- 流程、状态、开关类信息优先用 `Tag`
- 分类、渠道、入口、是或否等信息默认仍优先纯文本，避免把所有枚举值都做成彩色标签

## 不纳入典型页规则层的内容

下面这些信息保留为 HiUI 5 的通用设计系统背景，不直接纳入典型页生成强约束：

- 顶部导航、侧边导航的具体尺寸与折叠规则
- 品牌展示用字体说明和文档站展示策略
- 图表 / 装饰强调色的完整色阶
- 非典型页专用组件的完整皮肤矩阵

需要这类信息时，优先回 HiUI 5 原始组件文档或项目本地组件知识库，而不是在典型页规则层继续扩写。
