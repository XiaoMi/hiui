# HiUI Design 设计师入口

这份文档用设计协作语言解释 `hiui-design`。如果你只关心页面长什么样、哪里能改、交付是否合格，从这里开始即可。

## 它是什么

`hiui-design` 可以理解为一套后台页面的设计系统生产线。它不让 AI 随便画页面，而是先判断页面类型，再选择页面模具，最后用验收报告说明页面是否符合规范。

## 页面模具是什么

页面模具类似 Figma 里的标准页面模板。它规定页面的基本结构，例如：

- 页头放在哪里。
- 筛选区怎么排列。
- 表格和分页如何组合。
- 详情页分组如何组织。
- 数据看板如何安排指标、图表和说明。

AI 生成页面时，应先使用合适的模具，再替换业务内容。

## 哪些区域可以改

通常可以改的是业务槽位，例如：

- 页面标题和业务文案。
- 筛选字段。
- 表格列。
- 操作按钮。
- 详情字段。
- 图表指标和数据说明。

这些内容会根据具体业务变化。

## 哪些区域不应该随便改

不建议随便改的是规范锁定区域，例如：

- 页面壳和主体结构。
- 白色内容区。
- 页头、筛选区、表格、分页的层级关系。
- 页面滚动归属。
- 关键间距和基础视觉基线。
- 受管页面的 source marker 与 contract 边界。

这些区域决定页面是否仍属于同一套 HiUI 规范。

## 验收报告怎么看

最终报告会告诉你：

- 页面属于哪种类型。
- 使用了哪个页面模具。
- 哪些区域是按规范锁定的。
- 哪些内容是按业务替换的。
- preflight 是否通过。
- governance 是否通过。
- 是否还有设计风险或待确认问题。

如果报告是 blocked 或 failed，说明页面还不能当作正式交付。

## 设计师需要关注什么

- 页面类型是否符合业务场景。
- 页面结构是否符合预期。
- 可编辑区域是否覆盖了真实业务需求。
- 锁定区域是否被误改。
- 最终报告里的设计风险是否需要确认。

## 常见页面类型

当前重点支持：

- 普通表格页。
- 统计表格页。
- 树形表格页。
- 左树右表页。
- 数据看板页。
- 抽屉表单和抽屉详情。
- 全页编辑和全页详情。
- 异常反馈页。

后续新增页面类型时，应先沉淀成稳定模具，再进入团队正式使用。
