# 设计价值观

设计价值观，或设计理念，是设计师在设计过程中的主导思想，也是设计作品的灵魂和精髓，好的设计理念可以为设计师提供明晰的设计定位和目标，同时也可以为产品设计人员带来启发和引领。

HiUI 的设计价值观是以为用户带来舒适美好的用户体验为出发点、以产品体系化为目标而制定并坚持的。

### 模块化设计

由于中后台业务系统的逻辑复杂，要求组件具备更强大的功能和更丰富的类型，并且适用多种业务场景，因此，我们采用模块化的设计思路，将组件功能进行拆解且功能之间尽可能相互独立，用户可按照需求选用功能的同时不影响整体的结构；在模板设计上，我们采用功能菜单式模式，有效组织模板关系，提高模板的灵活性。

### 多样性

同一类型的组件和模板不能适用所有的场景和业务需求，因此，我们从使用场景、布局空间、信息层级等维度将组件和模板分为多种类型，用户可根据使用说明来选用适合业务的组件或模板。

### 布局

对于布局的体系化，从页面结构来说，定义几种常见的布局主题，用来解决不同复杂程度的信息结构，从布局结构、主题颜色和交互行为上形成体系化风格；从组件来说，应对多样化的界面元素，通过模板的形式，将同一种界面元素的布局位置定义到固定的位置，逐渐培养用户操作习惯，从而持续降低用户的学习门槛。

### 交互行为

对于交互行为的体系化，从页面来说，清晰定义不同页面类型之间的交互关系，比如列表页到详情页、详情页到编辑页的页面跳转关系以及反馈形式；从组件来说，按信息复杂度、信息量、操作效率、优先级等方面，定义多种场景下的组件，同时，遵循功能可见性、容错性、可读性、信噪比等原则来设计。

### 视觉风格

对于视觉风格的体系化，从页框架上定义了八种主题色，丰富却自成体系，足以让用户灵活满足个人偏好设置，以及系统的多种风格定位，同时，所有的组件都按同一种风格进行设计，运用到页面的任何位置，自动形成一致的视觉效果。
